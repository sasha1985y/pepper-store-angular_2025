### Техническое задание

Нужно создать интернет-магазин с каталогом товаров, корзиной, формой заказа, аутентификацией, управлением через сервисы и API.

## **1. Главная страница ProductList (`/`)**

- Отображение **списка товаров** (через сервис и `HttpClient`)
- Динамическая подгрузка данных из API (`/api/products`)
- Компоненты:
  - `ProductCardComponent` — карточка товара
  - `ProductListComponent` — список товаров
- Асинхронная загрузка данных через `async` pipe

**Темы для практики:**

- Основы компонентов 
- HTTP
- Структура и шаблоны 

## **2. Страница продукта (`/product/:id`)**

- Динамическая маршрутизация (`ActivatedRoute`)
- Загрузка данных через `HttpClient`
- Комментарии и отзывы (`Reactive Forms`)
- Возможность **добавить в корзину** (состояние через сервис)

**Темы для практики:**

- Маршрутизация
- Формы 
- Сервисы 

## **3. Корзина (`/cart`)**

- Хранение данных о товарах в **CartService**
- Использование `BehaviorSubject` для управления состоянием корзины
- Список товаров в корзине, сумма заказа
- Форма оформления заказа (Reactive Form + валидация)

**Темы для практики:**

- Сервисы и DI
- Формы и Reactive Forms

## **4. Аутентификация и личный кабинет (`/account`)**

- Вход и регистрация (фейковая аутентификация через сервис)
- Личный кабинет (`/account/:id`) с:
  - Историей заказов (API)
  - Формой изменения профиля (Reactive Form)

**Темы для практики:**

- Маршруты и защищённые страницы 
- Сервисы и DI 
- Формы 

## **5. API и серверная логика (`/api/`)**

- `/api/products` — получение всех товаров (GET)
- `/api/products/:id` — CRUD для товара (GET, POST, PUT, DELETE)
- `/api/cart` — обновление корзины (POST, DELETE)
- `/api/orders` — создание заказа

**Темы для практики:**

- API и HTTP 
- Сервисы 
- Маршруты 

## **Чеклист по функциональности:**

| №    | Функция               | Страница        | Ключевые технологии   |
| ---- | --------------------- | --------------- | --------------------- |
| 1    | Список товаров        | `/`             | HTTP, SVC, Async      |
